# Code of Conduct

Eddy is a project built for people whose nervous systems work differently. That shapes how we treat each other here.

---

## The short version

Be kind. Be patient. Assume good intent until you have clear evidence otherwise. Make space for people who communicate, work, and show up differently than you do.

---

## Who this applies to

This code of conduct applies to everyone participating in the Eddy project — in GitHub issues, pull requests, discussions, and any other community spaces associated with the project.

---

## What we expect

**Welcoming behaviour:**

- Using considerate, inclusive language
- Being patient with questions, especially from people new to Flutter, open source, or the codebase
- Accepting constructive feedback gracefully — and offering it the same way
- Respecting that people have different working styles, communication styles, and availability
- Acknowledging that neurodivergent contributors may process conversations, feedback, and delays differently — and that this is fine

**Unacceptable behaviour:**

- Harassment, insults, or dismissive language directed at any person
- Pressure tactics — urgency-shaming, calling out slow responses, demanding faster turnarounds
- Gatekeeping based on experience level, diagnosis, background, or anything else
- Sexualised language or unwanted attention of any kind
- Publicly sharing someone's private information without consent
- Dismissing or mocking someone for communicating in a non-standard way

---

## A note on neurodivergence

This project was created by and for neurodivergent people. ADHD, autism, anxiety, dyslexia, and other profiles are represented in our community. That means:

- Some contributors take longer to reply. That's not rudeness.
- Some contributors write in long bursts, then disappear for a while. That's not abandonment.
- Some contributors need very explicit, direct feedback. That's not a deficiency.
- Some contributors may need to re-ask something that was already discussed. That's not failure.

This community does not operate on neurotypical rhythms. Patience is not just nice to have here — it is part of how we work.

---

## Enforcement

If you experience or witness behaviour that violates this code of conduct, you can:

- Open a private discussion with the project maintainer via GitHub
- Send an email to the address in [SECURITY.md](SECURITY.md)

Reports will be handled with discretion. The goal is resolution, not punishment — but repeated or serious violations will result in removal from the community.

The maintainer reserves the right to remove, edit, or reject contributions, comments, or issues that violate this code of conduct.
